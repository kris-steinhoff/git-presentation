Git

Git is an open source, distributed version control system designed to handle everything from small to very large projects with speed and efficiency. Every Git clone is a full-fledged repository with complete history and full revision tracking capabilities, not dependent on network access or a central server. Branching and merging are fast and easy to do (http://git-scm.com/).

Creating a Repository

Turn your the current directory into a git repository: git init

Or, clone a remote repository: git clone git://github.com/ksteinhoff/git-presentation.git

Commiting Changes

Add files to your staging area: git add

Commit changes in your staging area to your repository: git commit

Commit all of the changes you've made to tracked files: git commit -a (files that are not already tracked must be added with "git add")

Pulling and Pushing

git pull will pull commits from the repo repository and merge the changes from the remote branch in to the your branch.

If you have write access to the remote repository you cloned from, you can push your commits to it: git push

If you're working in a cloned repository, the default branch (usually "master") will already be setup to work with "git push" and "git pull".

Branching

Create a new branch (based on your currently checked out branch) called "testing": git branch testing

Check out the "testing" branch: git checkout testing This changes files in your working directory.

Create a branch called "develop" that tracks the "develop" branch in remote repository "origin" and check it out: git checkout -b develop origin/develop

Merging

To merge the commits in the "testing" branch into the "develop", first checkout the "develop" branch: git checkout develop then do the merge: git merge testing
